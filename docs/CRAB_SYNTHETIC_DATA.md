# Crab Synthetic Data

The invasive species detector should be trained on all three printed crab
classes, even though the competition score needs the European green crab count.
That gives the model explicit negative examples for the two non-target species.

Generate a YOLO dataset from the three reference images:

```powershell
python -m tools.crab_generate_dataset --count 2000 --width 1280 --height 720
```

By default the tool looks for the reference crab images in `Downloads`, samples
real pool media from `Workspace\incoming`, and writes an ignored dataset under
`Workspace\datasets`. Synced stereo camera images under
`Workspace\incoming\pilot\...\stereo_sessions` are prioritized automatically
before the general incoming folder. Background discovery alternates across
sources so one large stereo folder does not crowd out every other source.

Useful options:

```powershell
python -m tools.crab_generate_dataset `
  --count 2000 `
  --background-dir .\Workspace\incoming\pilot `
  --green-template "C:\Users\Thoma\Downloads\European Green Crab Image (1).jpg" `
  --rock-template "C:\Users\Thoma\Downloads\Native Rock Crab (1).jpg" `
  --jonah-template "C:\Users\Thoma\Downloads\Jonah crab 2 (1).png"
```

The output contains:

- `data.yaml`: pass this to Ultralytics YOLO training.
- `images/train`, `images/val`: generated training and validation frames.
- `labels/train`, `labels/val`: YOLO box labels.
- `classes.txt`: class order.
- `manifest.json`: generation settings and class counts.
- `preview.jpg`: quick visual check with rendered boxes. This contact sheet is
  downsampled; inspect `images/train` or `images/val` when judging actual
  training-image sharpness.

The generator randomizes board pose, crab rotation/scale, corrugated board
texture, paper cutouts, tape/glare, underwater color cast, blur, noise,
vignetting, and JPEG compression. Real incoming camera media is used as the
background when available. By default it keeps almost the whole board in frame
and prevents meaningful crab overlap. It also mixes sparse, normal, and fuller
board layouts, with occasional larger crabs like the official sample boards.
Crab cutout colors are nudged darker and less saturated on average so they do
not train the detector only on vivid reference-image colors.
Scenes are also resampled when any labeled crab would render below the final
pixel-size floor, because those tiny labels are not useful for species learning.

For a wider but still realistic geometry pass with more camera-distance, roll,
and perspective variation:

```powershell
python -m tools.crab_generate_dataset `
  --count 2000 `
  --min-board-scale 0.52 `
  --max-board-scale 1.25 `
  --max-board-roll 68 `
  --min-board-tilt 0.18 `
  --max-board-tilt 0.92 `
  --board-center-jitter 0.20 `
  --board-corner-jitter 0.08 `
  --board-min-visible-fraction 0.91 `
  --background-limit 160 `
  --min-crab-scale 0.10 `
  --max-crab-scale 0.42 `
  --large-crab-fraction 0.32 `
  --large-crab-min-scale 0.34 `
  --large-crab-max-scale 0.52 `
  --min-crab-box-long-edge 40 `
  --sparse-layout-fraction 0.25 `
  --full-layout-fraction 0.30 `
  --even-placement-fraction 0.65 `
  --even-placement-jitter 0.38 `
  --max-crab-overlap 0.02 `
  --crab-spacing 0.015 `
  --camera-blur-fraction 0.12 `
  --max-camera-blur-sigma 0.45 `
  --jpeg-artifact-fraction 0.15 `
  --min-jpeg-quality 88
```

`--min-board-scale` and `--max-board-scale` control apparent camera distance:
small values create far-away boards, while larger values create closer boards.
`--max-board-tilt` controls how oblique the board can be relative to the camera.
Keep `--board-min-visible-fraction` high when the real task normally shows the
whole board. Lower it only when deliberately training rare cropped-board cases.

`--large-crab-fraction` controls how often the generator samples from the
larger printed-crab range. `--sparse-layout-fraction` and
`--full-layout-fraction` control how often the board has relatively low or high
crab coverage; the remaining boards use the normal layout profile.
Empty boards are not generated by default; use `--empty-fraction` only if you
want a small negative-example set.

`--even-placement-fraction` controls how often crabs are placed from a jittered
grid so the board has a more deliberate, evenly distributed layout. The
generator still keeps a random-placement share so the model sees both layout
styles.

The default blur and JPEG settings are intentionally moderate. Increase
`--camera-blur-fraction`, `--max-camera-blur-sigma`, or lower
`--min-jpeg-quality` only when you want a separate harsh-conditions dataset.
Use `--no-stereo-backgrounds` if you want to exclude auto-discovered stereo
camera images.

## Real-board plane generator

The more realistic dataset path uses real empty-board images as the scene and
projects crab cutouts into the manually annotated board plane. This is a better
starting point for closing the synthetic-to-real gap than relying only on fully
rendered board geometry.

Place empty-board photos under:

```text
Workspace\data\base images
```

Place additional real pool crab template images under:

```text
Workspace\data\real crabs
```

The template file names should include `green` or `euro`, `rock`, or `jonah`.
The generator also includes the MATE reference images from `Downloads` when they
are present, so real pool crab photos and the official reference cutouts are
used together.

Launch the GUI from the unified app with the `Crab Dataset` tab, or directly:

```powershell
python -m triton_analysis.apps.main_crab_dataset_generator
```

For each empty-board image, select the image and mark the four board corners in
order around the board. Save each plane. The annotations are written to:

```text
Workspace\data\board_plane_annotations.json
```

`Generate Dataset` writes a YOLO dataset under `Workspace\datasets`, with the
same `data.yaml`, `images/train`, `images/val`, `labels/train`, `labels/val`,
`classes.txt`, `manifest.json`, and `preview.jpg` structure as the original
synthetic generator.

The real-board generator defaults to a calmer appearance than the fully
synthetic renderer: crab color jitter is mild, no extra paper halo is added
around each crab, and the internal board texture is rendered at a faster 768 px
plane size before projection. Increase `Paper edge` only if the preview needs a
faint cutout/laminate border, and increase `Plane size` only when the generated
crabs look soft at your target training resolution.

## YOLO training

For a first synthetic-only detector pass, train a small pretrained YOLO model on
the generated dataset:

```powershell
.\.venv\Scripts\yolo.exe detect train `
  model=yolo26n.pt `
  data=.\Workspace\datasets\crab_synth_2000_steep_duller_20260608_181934\data.yaml `
  epochs=25 `
  imgsz=640 `
  batch=32 `
  device=0 `
  workers=4 `
  patience=8 `
  seed=60903 `
  project="$(Resolve-Path .\Workspace\runs\crab_yolo)" `
  name=synth2000_steep_duller_yolo26n_e25 `
  plots=True `
  amp=True
```

Run sample-board inference without adding those images to training:

```powershell
.\.venv\Scripts\yolo.exe detect predict `
  model=.\Workspace\runs\crab_yolo\synth2000_steep_duller_yolo26n_e25\weights\best.pt `
  source=.\Workspace\crab_mate_samples `
  imgsz=960 `
  conf=0.25 `
  iou=0.45 `
  device=0 `
  save=True `
  save_txt=True `
  save_conf=True `
  project="$(Resolve-Path .\Workspace\runs\crab_yolo)" `
  name=mate_samples_synth2000_yolo26n_conf025
```

Treat synthetic validation metrics as a renderer sanity check, not proof of
real-world performance. The MATE sample boards are currently the better quick
read on the synthetic-to-real gap.

For the current higher-quality synthetic-only pass, the useful checkpoint was
trained on:

- dataset:
  `Workspace\datasets\crab_synth_2000_hq_min40_1280_20260608_191925`
- model: `Workspace\models\yolo26m.pt`
- run:
  `Workspace\runs\crab_yolo\hq_min40_1280_yolo26m_e40_img1280_b4_restart`

The batch was reduced from 8 to 4 after an interrupted first attempt. On the
RTX 5090 laptop GPU this kept memory near 10.4 GB instead of about 18.7 GB.

```powershell
$project = Resolve-Path .\Workspace\runs\crab_yolo

.\.venv\Scripts\yolo.exe detect train `
  model=.\Workspace\models\yolo26m.pt `
  data=.\Workspace\datasets\crab_synth_2000_hq_min40_1280_20260608_191925\data.yaml `
  epochs=40 `
  imgsz=1280 `
  batch=4 `
  device=0 `
  workers=2 `
  patience=10 `
  seed=60928 `
  project="$project" `
  name=hq_min40_1280_yolo26m_e40_img1280_b4_restart `
  exist_ok=True `
  plots=True `
  cache=False `
  amp=True
```

Final synthetic validation metrics for that run:

- precision: 0.9985
- recall: 0.9955
- mAP50: 0.9950
- mAP50-95: 0.9911
